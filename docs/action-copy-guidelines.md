# Action Copy Guidelines

Use this standard for buttons, links, menus, and CTAs across Crewmodo.

## Principles

- Start actions with a verb: `Add`, `View`, `Edit`, `Send`, `Copy`, `Cancel`, `Mark`, `Request`.
- Name the object when ambiguity is possible: use `Add time`, `View estimate`, `Add change order`, not `Time`, `Estimate`, or `Change`.
- Keep visible buttons concise, but not cryptic. Two or three words is usually enough.
- Reserve primary/tonal visible buttons for the most likely next action on the screen.
- Put lower-frequency or administrative actions in a `More actions` overflow menu.
- Destructive actions should use explicit copy and appear last: `Cancel estimate`, `Void agreement`, `Delete cost`.
- Use the same phrase for the same action everywhere. Do not alternate between `Log time`, `Add time`, and `Time` for the same workflow unless the workflow is actually different.

## Preferred Common Actions

- `Add lead`
- `Create estimate`
- `View estimate`
- `Send estimate`
- `Send update`
- `Preview link`
- `Add time`
- `Add cost`
- `Add change order`
- `Send for approval`
- `Request review`
- `Mark job complete`
- `Copy approval link`
- `Cancel estimate`
- `Void agreement`

## Button Hierarchy

- Primary: the main submit or progression action, such as `Send estimate`.
- Tonal/secondary: common but non-final actions, such as `Add time`.
- Text: lower-emphasis actions in dense lists or menus.
- Icon-only: universally recognized actions or overflow; always include `aria-label` and `title`.
